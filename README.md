# 🛰️ Covenant Sentinel

> **Sentinel watches. The Skeptic doubts. You decide.**

An enterprise credit-monitoring agent built for the **RAISE Summit Hackathon 2026 — Vultr track**. When a quarterly filing arrives, Sentinel wakes up on its own: it plans an investigation, reads the executed facility documents, **recalculates covenant ratios with a deterministic tool**, hunts the causes in the transaction ledger — then must **defend its memo against The Skeptic**, an adversarial internal reviewer running on a different model, before any human sees it. The final output is an escalation-grade memo with verbatim, click-through citations, a **computed** confidence score, and the exact **cost of the run**.

Built entirely during the event (July 4–5, 2026). Solo team.

## Three scenarios, three different verdicts — the agent decides

The inbox offers three filings. Same agent, same prompts, no scripting — the outcome depends on what the agent finds:

| Filing | What actually happened | Agent's verdict |
|---|---|---|
| **Q2-2025** | Routine quarter, leverage 3.13x vs 3.50x | ✅ **OK** — no escalation |
| **Q3-2025** | Leverage 3.70x > 3.50x… but Amendment No. 2 grants a covenant holiday at 4.00x | ⚠️ **EARLY WARNING** — the twist |
| **Q4-2025** | Leverage 4.41x — above even the amended 4.00x ceiling, liquidity below the €3m floor | 🚨 **BREACH** — genuine default |

The Q3 story is the heart of the demo: first draft says **BREACH**, The Skeptic objects (*"you never checked the amendments"*), Sentinel re-investigates, finds the covenant holiday, and revises to **EARLY WARNING** — live, in the trace.

## Four Vultr models, each with a job

| Role | Model | Why |
|---|---|---|
| 🚦 Triage | Nemotron-3-Nano ($0.13/1M) | Cheapest model gates the expensive ones — archives noise before waking the agent |
| 🛰️ Sentinel | Kimi-K2.6 | Plans, investigates, drafts — strong agentic reasoning |
| ⚔️ The Skeptic | DeepSeek-V4-Flash | Adversarial review on a **different brain** — no shared blind spots |
| 🔎 Retrieval | VultronRetrieverFlash | Semantic rerank over the credit file (keyword BM25 fallback) |

An independent fifth model (Qwen3.6-27B) serves as **LLM-as-judge** in the eval harness — deliberately absent from the production pipeline.

## The eval harness (`/eval`)

The corpus is generated by script with **known ground truth**, so every run is scored objectively, not by vibes:

- ✔ correct verdict (OK / EARLY_WARNING / BREACH)
- ✔ exact leverage recomputed via `calc_ratio` (3.13x / 3.70x / 4.41x)
- ✔ every citation verified verbatim against its source page
- ✔ Amendment No. 2 found and used where it matters
- ✔ all material one-off movements carried in the memo
- ✔ Skeptic review completed and approved

Plus an **LLM-as-judge** pass scoring faithfulness & completeness of the final memo against its cited sources — on Vultr.

## Why this isn't "basic RAG"

- **Event-driven, not chat-driven** — the agent reacts to document arrival, no prompt needed.
- **Plans, then retrieves multiple times** — including a second retrieval wave forced by the Skeptic's challenge.
- **Calls deterministic tools** — ratios come from `calc_ratio`, transactions from `query_ledger`; the model never does financial math.
- **Makes decisions** — BREACH vs EARLY_WARNING vs OK, escalation path, what stays unexplained.
- **Audits itself** — adversarial Skeptic review with objection/resolution loop, visible in the trace.
- **Code-enforced honesty** — a materiality sweep force-lists any material movement the model failed to attribute, and lowers the confidence score accordingly.
- **Verifiable output** — every citation is checked verbatim against the source page in code; confidence is a published formula; each run reports its real dollar cost from the Vultr price list.

## Architecture

```
Next.js (App Router, TS) — modular monolith, one deploy
├── src/config/app.config.ts   ← every knob: providers, 5 models, thresholds, prices, weights
├── src/lib/llm/               ← OpenAI SDK → Vultr Serverless Inference (+ mock provider, + per-run cost tracker)
├── src/lib/agent/             ← hand-rolled loop: triage → plan → investigate → draft
│                                 → Skeptic review → resolve → confidence
├── src/lib/tools/             ← search_documents · read_page · calc_ratio · query_ledger
├── src/lib/eval/              ← ground-truth scorecard + LLM-as-judge
├── src/app/api/agent/run      ← SSE stream of every agent event (live trace + per-scenario replay)
├── src/app/api/eval           ← scorecard + judge endpoints
├── src/components/            ← 3-panel workbench: inbox · trace · memo + page viewer + /eval board
├── src/scripts/generate-data.ts ← generates the whole corpus (run during the event)
└── deploy/                    ← one-command Vultr compute deployment (cloud-init + API script)
```

- **LLM**: [Vultr Serverless Inference](https://docs.vultr.com/products/compute/serverless-inference), OpenAI-compatible API.
- **Agent loop**: hand-rolled JSON-mode ReAct — no framework, full control, easy to stream.
- **Corpus**: fully simulated credit file (12-page facilities agreement, 2 amendments, 6 quarters of statements, compliance certificates, 3 arrival filings, ~1,400-row ledger) generated by a seeded script.

## Run it

```bash
npm install
node src/scripts/generate-data.ts   # generate the corpus
cp .env.example .env.local          # paste your Vultr API key (empty = mock mode)
npm run dev
```

Open http://localhost:3000 and pick a filing in the inbox. Without an API key the app runs in **mock mode** (scripted Q3 storyline) — with a key everything runs live on Vultr.

Each successful live run is persisted; the **▶** button replays it instantly with realistic pacing, zero API calls — demo insurance.

## Deploy on Vultr (everything on Vultr)

```bash
VULTR_API_KEY=<account api key> ./deploy/vultr-deploy.sh
```

Creates a compute instance (Paris), clones this repo, builds, and serves on port 80 — inference *and* hosting on Vultr.
