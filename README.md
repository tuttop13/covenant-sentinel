# 🛰️ Covenant Sentinel

> **Sentinel watches. The Skeptic doubts. You decide.**

An enterprise credit-monitoring agent built for the **RAISE Summit Hackathon 2026 — Vultr track**. When a quarterly filing arrives, Sentinel wakes up on its own: it plans an investigation, reads the executed facility documents, **recalculates covenant ratios with a deterministic tool**, hunts the causes in the transaction ledger — then must **defend its memo against The Skeptic**, an adversarial internal reviewer, before any human sees it. The final output is an escalation-grade memo with verbatim, click-through citations and a **computed** confidence score.

Built entirely during the event (July 4–5, 2026). Solo team.

## The demo story (90 seconds)

1. 📬 The **Q3-2025 filing** for Meridian Logistics SA lands in the inbox — Sentinel wakes up.
2. It reads the covenant clause (§7.1(a): leverage ≤ 3.50x), recomputes the ratio with `calc_ratio`: **3.70x — apparent BREACH**.
3. It queries the ledger and matches the causes: a €1.2m litigation settlement + a €2.8m revolver-funded fleet purchase.
4. Draft memo: **BREACH**. ⚔️ **The Skeptic objects**: *"You never checked whether an amendment modifies the threshold for this period."*
5. Sentinel re-retrieves → finds **Amendment No. 2**: a covenant holiday raising the ceiling to 4.00x for Q3/Q4-2025.
6. Revised memo: **EARLY WARNING** (compliant now, but above the threshold that reverts in Q1-2026) — approved by The Skeptic, **confidence 85%**, every citation opening the exact page and highlighted clause.

## Why this isn't "basic RAG"

- **Event-driven, not chat-driven** — the agent reacts to document arrival, no prompt needed.
- **Plans, then retrieves multiple times** — including a second retrieval wave forced by the Skeptic's challenge.
- **Calls deterministic tools** — ratios come from `calc_ratio`, transactions from `query_ledger`; the model never does financial math.
- **Makes decisions** — BREACH vs EARLY_WARNING vs OK, escalation path, what stays unexplained.
- **Audits itself** — adversarial Skeptic review with objection/resolution loop, visible in the trace.
- **Verifiable output** — every citation is checked verbatim against the source page in code; confidence is a published formula, not a vibe.

## Architecture

```
Next.js (App Router, TS) — modular monolith, one deploy
├── src/config/app.config.ts   ← every knob: provider, model, thresholds, weights
├── src/lib/llm/               ← OpenAI SDK → Vultr Serverless Inference (+ mock provider)
├── src/lib/agent/             ← hand-rolled loop: plan → investigate → draft
│                                 → Skeptic review → resolve → confidence
├── src/lib/tools/             ← search_documents · read_page · calc_ratio · query_ledger
├── src/app/api/agent/run      ← SSE stream of every agent event (the live trace)
├── src/components/            ← 3-panel workbench: inbox · trace · memo + page viewer
└── src/scripts/generate-data.ts ← generates the whole corpus (run during the event)
```

- **LLM**: [Vultr Serverless Inference](https://docs.vultr.com/products/compute/serverless-inference) (`kimi-k2-instruct`), OpenAI-compatible API.
- **Agent loop**: hand-rolled JSON-mode ReAct — no framework, full control, easy to stream.
- **Corpus**: fully simulated credit file (12-page facilities agreement, 2 amendments, 8 quarters of statements, compliance certificates, 504-row ledger) generated by a seeded script.

## Run it

```bash
npm install
node src/scripts/generate-data.ts   # generate the corpus
cp .env.example .env.local          # paste your Vultr API key (empty = mock mode)
npm run dev
```

Open http://localhost:3000 and click **"📬 Simulate arrival: Q3-2025 filing"**.

Without an API key the app runs in **mock mode** (scripted agent, same storyline) — with a key it runs live on Vultr Serverless Inference.
